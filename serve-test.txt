To test the flasher from the NAS content over HTTP instead of file://, serve this folder with a local web server.

Example from the Pi:

cd /mnt/clu-nas/flash
python3 -m http.server 8037

Then open:
http://192.168.1.232:8037/

Why:
- manifest.json and binary fetches often fail from file:// URLs
- WebUSB pages should be tested over HTTP/HTTPS, not direct filesystem browsing

Notes:
- On LAN HTTP, Chromium may still restrict some device features depending on origin/security rules.
- GitHub Pages / HTTPS remains the better final test target.
